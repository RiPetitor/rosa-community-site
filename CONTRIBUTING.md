# Руководство для авторов

Добро пожаловать! Это руководство поможет вам создавать статьи для блога и страницы документации на сайте ROSA Community.

## Быстрый старт

### Структура проекта

```
content/
├── blog/           # Статьи блога
│   └── my-post.md
└── docs/           # Документация
    ├── _index.md   # Главная страница документации
    └── getting-started/
        ├── _index.md       # Раздел
        └── installation.md # Страница
```

### Как внести изменения

1. **Через GitHub** — нажмите кнопку «Внести правки» внизу любой страницы
2. **Локально** — клонируйте репозиторий и запустите `zola serve`

## Написание статей для блога

Создайте файл в папке `content/blog/` с расширением `.md`.

### Базовый шаблон

```markdown
+++
title = "Заголовок статьи"
date = 2026-01-30
description = "Краткое описание для превью в списке статей"

[taxonomies]
categories = ["Новости"]
tags = ["rosa", "обновление"]

[extra]
author = "Ваше имя"
+++

Вводный абзац статьи, который будет показан в списке.

<!-- more -->

## Основной контент

Здесь начинается полный текст статьи...
```

### Поля front matter

| Поле | Обязательное | Описание |
|------|--------------|----------|
| `title` | Да | Заголовок статьи |
| `date` | Да | Дата публикации (ГГГГ-ММ-ДД) |
| `description` | Рекомендуется | Краткое описание для SEO и превью |
| `categories` | Рекомендуется | Рубрика: Новости, Руководства, Обзоры |
| `tags` | Рекомендуется | Теги через запятую |
| `extra.author` | Рекомендуется | Имя автора |

### Дополнительные возможности

```toml
draft = true                    # Черновик (не публикуется)
slug = "custom-url"             # Свой URL вместо имени файла
aliases = ["/old-url/"]         # Редирект со старого адреса
extra.featured = true           # Пометка «Передовица»
```

## Написание документации

Документация организована в разделы (папки) и страницы (файлы).

### Создание страницы

Создайте файл `.md` в нужном разделе `content/docs/`.

```markdown
+++
title = "Установка ROSA Linux"
description = "Пошаговая инструкция по установке системы"
weight = 1
+++

Текст документации...
```

### Создание раздела

Создайте папку и файл `_index.md` внутри неё.

```markdown
+++
title = "Начало работы"
description = "Первые шаги с ROSA Linux"
template = "docs-section.html"
weight = 1
sort_by = "weight"
+++

Описание раздела (необязательно).
```

### Поля front matter

| Поле | Описание |
|------|----------|
| `title` | Название страницы/раздела |
| `description` | Краткое описание |
| `weight` | Порядок сортировки (меньше = выше) |
| `template` | Шаблон: `docs-section.html` для разделов |

## Оформление контента

### Заголовки

Используйте заголовки `##` и `###` для структурирования. Они автоматически попадут в оглавление справа.

```markdown
## Раздел

### Подраздел

#### Мелкий подраздел
```

### Блоки-подсказки (Callouts)

```html
<div class="tip">
  <div class="title">Совет</div>
  Полезная информация для пользователя.
</div>

<div class="info">
  <div class="title">Примечание</div>
  Дополнительные сведения.
</div>

<div class="warning">
  <div class="title">Внимание</div>
  Предупреждение о возможных проблемах.
</div>

<div class="danger">
  <div class="title">Осторожно</div>
  Критически важная информация.
</div>
```

### Код

Используйте тройные обратные кавычки с указанием языка:

````markdown
```bash
sudo dnf update
sudo dnf install package-name
```

```python
print("Hello, ROSA!")
```
````

### Таблицы

```markdown
| Команда | Описание |
|---------|----------|
| `dnf update` | Обновить систему |
| `dnf install` | Установить пакет |
| `dnf remove` | Удалить пакет |
```

## Локальная разработка

### Требования

- [Zola](https://www.getzola.org/documentation/getting-started/installation/) — генератор статических сайтов

### Команды

```bash
# Клонировать репозиторий
git clone https://github.com/RiPetitor/rosa-community-site.git
cd rosa-community-site

# Запустить локальный сервер
zola serve

# Сайт доступен по адресу http://127.0.0.1:1111
```

### Проверка перед отправкой

```bash
# Проверить сборку
zola build

# Проверить на ошибки
zola check
```

## Отправка изменений

### Через Pull Request

1. Сделайте форк репозитория
2. Создайте ветку для изменений
3. Внесите правки
4. Отправьте Pull Request

### Через кнопку «Внести правки»

На каждой странице документации и блога есть кнопка «Внести правки». Она откроет редактор GitHub, где вы сможете предложить изменения.
