# Руководство для авторов

Добро пожаловать в редакцию! Это руководство поможет вам создавать статьи для блога и страницы документации.

## Быстрый старт

### Структура редакции

```
content/
├── blog/           # Передовицы и заметки
│   └── my-post.md
└── docs/           # Справочные материалы
    ├── _index.md   # Главная страница документации
    └── getting-started/
        ├── _index.md       # Раздел
        └── installation.md # Страница
```

### Как внести правки

1. **Через GitHub** — нажмите кнопку «Внести правки» внизу любой страницы
2. **Создать новое** — используйте кнопки «+ Написать» (блог) или «+ Добавить страницу» (документация)
3. **Локально** — клонируйте репозиторий и запустите `zola serve`

## Написание статей для блога

Создайте файл в папке `content/blog/` с расширением `.md`.

### Базовый шаблон

```markdown
+++
title = "Заголовок статьи"
date = 2026-01-30
description = "Краткое описание для превью"

[taxonomies]
categories = ["Новости"]
tags = ["rosa", "обновление"]

[extra]
author = "Ваше имя"
+++

Вводный абзац статьи — будет показан в списке.

<!-- more -->

## Основной текст

Здесь начинается полный текст статьи...
```

### Поля заголовка

| Поле | Обязательное | Описание |
|------|--------------|----------|
| `title` | Да | Заголовок статьи |
| `date` | Да | Дата публикации (ГГГГ-ММ-ДД) |
| `description` | Рекомендуется | Краткое описание для SEO и превью |
| `categories` | Рекомендуется | Рубрика: Новости, Руководства, Обзоры |
| `tags` | Рекомендуется | Теги через запятую |
| `extra.author` | Рекомендуется | Имя автора |

### Дополнительные возможности

```toml
draft = true                    # Черновик (не публикуется)
slug = "custom-url"             # Свой URL вместо имени файла
aliases = ["/old-url/"]         # Редирект со старого адреса
extra.featured = true           # Пометка «Передовица»
```

## Написание документации

Документация организована в разделы (папки) и страницы (файлы).

### Создание страницы

Создайте файл `.md` в нужном разделе `content/docs/`.

```markdown
+++
title = "Установка ROSA Linux"
description = "Пошаговая инструкция"
weight = 1
+++

{{ doc_dates() }}

Текст документации...
```

> Важно: для страниц документации обязательно добавляйте `{{ doc_dates() }}` сразу после заголовка.  
> Это включит автоматические даты «Опубликовано» и «Обновлено».  
> Для файлов `_index.md` (страницы разделов) блок не нужен.

### Создание раздела

Создайте папку и файл `_index.md` внутри неё.

```markdown
+++
title = "Начало работы"
description = "Первые шаги с ROSA Linux"
template = "docs-section.html"
weight = 1
sort_by = "weight"
+++

Описание раздела (необязательно).
```

### Поля заголовка

| Поле | Описание |
|------|----------|
| `title` | Название страницы/раздела |
| `description` | Краткое описание |
| `weight` | Порядок сортировки (меньше = выше) |
| `template` | Шаблон: `docs-section.html` для разделов |

## Оформление текста

### Заголовки

Используйте `##` и `###` для структурирования. Они автоматически попадут в оглавление.

```markdown
## Раздел

### Подраздел

#### Мелкий подраздел
```

### Блоки-подсказки

```html
<div class="tip">
  <div class="title">Совет</div>
  Полезная информация.
</div>

<div class="info">
  <div class="title">Примечание</div>
  Дополнительные сведения.
</div>

<div class="warning">
  <div class="title">Внимание</div>
  Предупреждение.
</div>

<div class="danger">
  <div class="title">Осторожно</div>
  Критически важно.
</div>
```

### Код

````markdown
```bash
sudo dnf update
sudo dnf install package-name
```
````

### Таблицы

```markdown
| Команда | Описание |
|---------|----------|
| `dnf update` | Обновить систему |
| `dnf install` | Установить пакет |
```

## Локальная работа

### Требования

- [Zola](https://www.getzola.org/documentation/getting-started/installation/) — генератор статических сайтов

### Команды

```bash
# Клонировать репозиторий
git clone https://github.com/RiPetitor/rosa-community-site.git
cd rosa-community-site

# Запустить локальный сервер
zola serve

# Газета доступна по адресу http://127.0.0.1:1111
```

### Проверка перед отправкой

```bash
zola build   # Проверить сборку
zola check   # Проверить на ошибки
```

## Отправка материалов

### Через Pull Request

1. Сделайте форк репозитория
2. Создайте ветку для изменений
3. Внесите правки
4. Отправьте Pull Request

### Через кнопки на сайте

- **«Внести правки»** — редактировать существующую страницу
- **«+ Написать»** — создать новую статью в блоге
- **«+ Добавить страницу»** — создать новую страницу документации
