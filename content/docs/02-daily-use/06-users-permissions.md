Linux — многопользовательская система. Понимание прав доступа важно для безопасности.

## Типы пользователей

- **root** — суперпользователь с полным доступом.
- **Обычные пользователи** — ограниченные права.
- **Системные пользователи** — для служб и демонов.

<div class="danger">
  <div class="title">Опасно</div>
  Не работайте постоянно под root. Используйте <code>sudo</code> для административных задач.
</div>

## Управление пользователями

### Графический способ

1. Откройте «Параметры системы» → «Пользователи».
2. Добавьте, измените или удалите учётную запись.

### Через терминал

#### Создание пользователя

```bash
sudo useradd -m -s /bin/bash username
sudo passwd username
```

#### Удаление пользователя

```bash
sudo userdel -r username
```

#### Добавление в группу

```bash
sudo usermod -aG группа username
```

#### Список групп пользователя

```bash
groups username
```

## Важные группы

| Группа | Назначение |
| --- | --- |
| `wheel` | Право использовать sudo |
| `audio` | Доступ к звуковым устройствам |
| `video` | Доступ к видеоустройствам |
| `docker` | Запуск Docker без sudo |
| `libvirt` | Управление виртуальными машинами |

### Добавление права sudo

```bash
sudo usermod -aG wheel username
```

## Права доступа к файлам

Каждый файл имеет владельца, группу и права доступа.

### Просмотр прав

```bash
ls -la
```

Вывод: `-rwxr-xr-x 1 user group 1234 Jan 1 12:00 file`

- `r` — чтение (read)
- `w` — запись (write)
- `x` — выполнение (execute)

### Изменение прав

```bash
# Числовой формат (755 = rwxr-xr-x)
chmod 755 file

# Символьный формат
chmod u+x file     # добавить выполнение владельцу
chmod go-w file    # убрать запись у группы и остальных
```

### Изменение владельца

```bash
sudo chown user:group file
sudo chown -R user:group directory/
```

## sudo

Команда `sudo` позволяет выполнять действия от имени root.

### Использование

```bash
sudo команда
```

### Редактирование файлов от root

```bash
sudo nano /etc/hosts
```

### Настройка sudo

Файл `/etc/sudoers` редактируется через:

```bash
sudo visudo
```

<div class="warning">
  <div class="title">Предупреждение</div>
  Неправильное редактирование sudoers может заблокировать доступ к sudo.
</div>

## Смена пароля

### Свой пароль

```bash
passwd
```

### Пароль другого пользователя

```bash
sudo passwd username
```

## Блокировка учётной записи

```bash
# Заблокировать
sudo usermod -L username

# Разблокировать
sudo usermod -U username
```
