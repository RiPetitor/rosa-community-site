Если система не загружается, не паникуйте. Большинство проблем можно решить.

## Чёрный экран после BIOS

### Причины

- Повреждённый загрузчик GRUB
- Проблемы с драйвером видеокарты
- Повреждённое ядро

### Решения

#### Попробуйте предыдущее ядро

1. При загрузке нажмите `Esc` или `Shift` для вызова меню GRUB.
2. Выберите «Advanced options for ROSA Linux».
3. Выберите предыдущую версию ядра.

#### Загрузка в безопасном режиме

1. В меню GRUB нажмите `e` на пункте загрузки.
2. Найдите строку, начинающуюся с `linux`.
3. Добавьте в конец: `nomodeset`
4. Нажмите `Ctrl + X` для загрузки.

<div class="info">
  <div class="title">Параметр nomodeset</div>
  Отключает драйвер видеокарты на этапе загрузки. Полезно при проблемах с NVIDIA/AMD.
</div>

## GRUB не появляется

### Восстановление GRUB

Загрузитесь с Live USB ROSA Linux:

```bash
# Определите раздел с системой
lsblk

# Примонтируйте корневой раздел
sudo mount /dev/sdaX /mnt

# Для UEFI примонтируйте EFI-раздел
sudo mount /dev/sdaY /mnt/boot/efi

# Войдите в систему
sudo chroot /mnt

# Переустановите GRUB (UEFI)
grub2-install --target=x86_64-efi --efi-directory=/boot/efi

# Или для BIOS
grub2-install /dev/sda

# Обновите конфигурацию
grub2-mkconfig -o /boot/grub2/grub.cfg

# Выйдите
exit
sudo reboot
```

## Kernel panic

Сообщение «Kernel panic — not syncing» означает критическую ошибку ядра.

### Причины

- Повреждённые системные файлы
- Несовместимое оборудование
- Ошибки файловой системы

### Решение

1. Загрузитесь с предыдущим ядром.
2. Проверьте файловую систему:
   ```bash
   sudo fsck /dev/sdaX
   ```
3. Переустановите ядро:
   ```bash
   sudo dnf reinstall kernel
   ```

## Зависание на логотипе

### Проверка режима загрузки

Добавьте параметры для диагностики:

1. В GRUB нажмите `e`.
2. Уберите `quiet splash` из строки `linux`.
3. Загрузитесь и наблюдайте сообщения.

### Частые причины

- **Драйвер NVIDIA** — попробуйте `nomodeset`
- **USB-устройства** — отключите всё лишнее
- **Сетевой интерфейс** — если зависает на NetworkManager

## Восстановление из Live USB

С Live-образа можно:

1. Скопировать важные данные
2. Исправить конфигурацию
3. Переустановить пакеты
4. Восстановить загрузчик

```bash
# Монтирование системы
sudo mount /dev/sdaX /mnt
sudo mount --bind /dev /mnt/dev
sudo mount --bind /proc /mnt/proc
sudo mount --bind /sys /mnt/sys

# Вход в систему
sudo chroot /mnt

# Теперь можно выполнять команды как в установленной системе
```

<div class="tip">
  <div class="title">Совет</div>
  Всегда держите под рукой Live USB с ROSA Linux для восстановления.
</div>
